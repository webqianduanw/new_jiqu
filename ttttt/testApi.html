执行原生操作

JSBridge.invoke(action,params,callback);
  • action: 执行操作标识，必须提供，字符串形式， 操作对应见下

  • params: 执行操作所附带参数，可选提供，js对象，例如

    {
    num1: 1,
    text1: 'kk',
    array1: [1, 2, 3, 4],
    array2: ['a', 'b', 'c'],
    obj1: {
        aaa: "bbb",
        ccc: 1111
    }
}

  • callback(即callId): 执行完成后的回调函数（如果有），用于下文 移动端回调时传回

setTitle 设置窗口标题栏

示例 : JSBridge.invoke({'action':'setTitle','params':{'title':'这是导航'}})

  • 参数：

    {
    title: ""//标题
}

  • 回调：无

open 打开原生功能页面

示例 : JSBridge.invoke({'action':'open','params':{}})

  • 参数：

    {
    url: "https://www.baidu.com", //链接地址，与下方功能页面标识page 2选一
    page: "app_home",   //相应功能页面标识，各功能页面对应名称若ios和安卓需要尽量达成一致
    data: {             //可选，如果提供则传给相应功能页面
        aaa: "bbb"
    }
}

  • 回调：无

getUser 获取当前用户信息

  • 参数：无

  • 回调：

    {
    accountId:12344, //账户ID
    nickName:'昵称1234' //昵称
}

原生执行JS操作

如果存在回调关系 


  nativeCallBack(callId,callback);

callId: 从前端获得并传回

callback: 执行完成后的回调函数（如果有），包含一个返回对象参数，除业务本身返回数据外，该对象还必须包含属性 resultCode 其值必须为 success cancel fail 三选一；若需要可包含错误代码 errorCode 及错误描述信息 errorMessage； errorCode 采用数字类型，ios和安卓实现操作时需要达成约定，相同错误采用相同编号；在失败时 errorCode 不可以为 0；下方文档所示回调数据除特殊指出外均默认包含左侧提及的公共参数

如果不存在回调关系 


nativeCallJS(params)

params: 结构如下


    {
      action: "reload",         //必选
      data: {             //可选，如果提供则传给相应功能页面
          aaa: "bbb"
      }
  }
